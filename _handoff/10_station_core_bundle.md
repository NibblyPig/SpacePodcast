```markdown
# Orbital Station Elysium — Structural Overview

Elysium is a large-scale modular orbital megastructure built around a central non-rotating spine, with multiple rotating rings attached at intervals.

The station operates in orbit around a ruined temperate-origin world, under a dim star providing cyclical solar exposure.

---

## Primary Components

- Central Spine (non-rotating)
- Rotating Rings (8 primary)
- Spokes (rotating structural connectors)
- Ring Hubs
- External Trusses
- Solar Arrays
- Radiator Arrays
- Transmission Beacon Assembly

---

## Structural Model

- Spine = fixed backbone
- Rings = rotating districts
- Spokes = rotating connectors
- External systems = non-rotating

---

## Power Model

- Solar arrays mounted on dorsal/ventral trusses
- Power varies by exposure cycle
- High-power and low-power operational phases

---

## Design Philosophy

- Incremental expansion over time
- Mixed-use rings with dominant roles
- Continuous automated operation
```

```markdown
# Ring Architecture

## Overview

Rings are rotating mixed-use districts providing artificial gravity.

---

## Structure

Each ring includes:

- rotating hub
- spokes (6–12)
- inner service band
- main inhabited band (outer circumference)

---

## Gravity

- increases with radius
- highest at outer ring
- lowest near hub

---

## Layout Bands

- hub (low gravity)
- spoke corridors
- service decks
- habitation / primary use decks

---

## Notes

Rings are not single-purpose.

Each contains mixed-use zoning with a dominant function.
```

Obsidian/04_Spoke_and_Hub_Design.md

```markdown
# Spoke and Hub Design

## Hub System

Each ring connects via:

- non-rotating spine node
- rotating ring hub
- transit couplings

---

## Transit Couplings

Allow pods to transfer between spine and rotating rings.

- external to spine
- isolate track segment
- rotate to match ring
- align with spoke track

---

## Spokes

- rotating with ring
- 6–12 per ring
- carry transit + utilities

---

## Gravity Gradient

- hub: near zero-g
- spoke: increasing
- ring edge: full gravity
```

```markdown
# Elysium Documentation Guide

## Core Principle

Maintain spatial consistency without excessive detail.

Track only: - where something is (ring → quarter) - what type of place
it is

Do not track: - exact layouts - adjacency - floorplans

------------------------------------------------------------------------

## Structure Overview

Elysium/ Rings/ \[Ring\]/ Overview.md Q1.md Q2.md Q3.md Q4.md

Locations/ \[Generic Types\].md Named/ [Named
Locations](#named-locations).md

Episodes/ Season 1/ E01/ script.md refs.md

------------------------------------------------------------------------

## Two Types of Locations

### 1. Discrete Locations (numbered)

Examples: - Hydroponics 2 - Storage 3

These exist inside quarters.

------------------------------------------------------------------------

### 2. Ring-Wide Systems (not numbered)

Examples: - Administration - Security - Power Distribution

These exist only in Overview.md

------------------------------------------------------------------------

## Quarter Files

Each QX.md contains:

-   list of location types only
-   no layout or adjacency

------------------------------------------------------------------------

## Named Locations

Stored in: Locations/Named/

Each must include:

## Meta

-   Ring: Delta
-   Quarter: Q2
-   Type: Hydroponics
-   Status: Operational
-   First Referenced: S01E01

------------------------------------------------------------------------

## Naming Convention

Use: \[Location Type\] + Number

Examples: - Hydroponics 2 - Habitation 5

------------------------------------------------------------------------

## Episode References

refs.md format:

## Locations Used

-   Delta Ring --- Q2 --- Hydroponics 2

## Events

-   short factual notes

## Persistent Notes

-   only things worth remembering

------------------------------------------------------------------------

## Continuity Rules

Track only: - recurring values - ongoing issues - unchanged states

Avoid: - over-logging - resolving everything

------------------------------------------------------------------------

## Writing Constraints

Avoid: - exact positioning - adjacency - distances

Use: - "within" - "across" - "in Quarter X"

------------------------------------------------------------------------

## Status System

Optional but recommended:

-   Operational
-   Degraded
-   Offline
-   Under Maintenance

------------------------------------------------------------------------

## Final Principle

The system should feel like: a quiet logbook, not a simulation.
```
